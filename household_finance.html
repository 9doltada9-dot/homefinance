Version History
หลักการ Versioning: v[major].[minor].[patch]
· major — เปลี่ยนโครงสร้างใหญ่ / เปลี่ยน database schema
· minor — เพิ่ม feature ใหม่ / เปลี่ยน UI หลัก
· patch — แก้ bug / ปรับ UI เล็กน้อย
v2.2.0 — เพิ่ม: ประเภทรายการ "โอน/ฝาก" (transfer) — ไม่นับในยอดรายรับ/รายจ่าย, แสดง badge สีฟ้าในรายการ, รองรับทั้ง form เพิ่ม/แก้ไขและ filter
v2.1.0 — เพิ่ม: รายการในหมวด, เลือก desc จาก DB, วันที่ภาษาไทย, นาฬิกาในแถบบน, ซ่อนยอดเงินเดือน, ยกเลิกการลบ, version history
v2.0.0 — เปลี่ยนจาก Google Sheets เป็น Supabase, Realtime connection monitor, categories table แบบ relationship
v1.0.0 — เวอร์ชันเริ่มต้น: บันทึกรายรับ/รายจ่าย, Settlement, Dashboard, Google Sheets export

สำรองข้อมูลออกไฟล์
วิธีตั้งค่า Supabase (3 ขั้นตอน)
1
สร้างโปรเจกต์ที่ supabase.com (ฟรี) → กด New project → ตั้งชื่อ เช่น homefinance → จดรหัสผ่านฐานข้อมูลไว้
2
สร้าง Table → เปิด SQL Editor → วาง SQL ด้านล่าง → Run
create table transactions ( id text primary key, date text, type text, cat text, desc text, amt numeric, person text, split boolean, status text, note text, created_at timestamptz default now() ); alter table transactions enable row level security; create policy "public read" on transactions for select using (true); create policy "public write" on transactions for all using (true);
3
Copy credentials → ไปที่ Project Settings → API → Copy Project URL และ anon public key → วางในช่องด้านบน → กดทดสอบ ✓
